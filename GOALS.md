# Project Goals

## Local Packaging Tooling

An engineer needs to build software locally in a straightforward
way without relying on hosted services.

## Shareability

Packagers want to create a base static build environment that is shareable
among their packaging team, business partners or customers.

## Content Delivery

Entities or groups want to deliver their packaged software into operating systems
in a standard way, allowing third parties to do the same, be it closed or
open-sourced software which includes the possibility of self-hosting such content.

## Content Integrity

Prodsec engineers want a standard way to certify that a collection of packages
can be assertable and certifiable in an automated way
to assert the delivered content is what it claims to be.

## Extendable

Packagers want the ability to handle different build domains wihtin the same tool,
such as RPM building, raw OCI layer delivery and so on.

This mean they should also able to extend the tool by creating their own
build plugins.
